# FL0024: Expression-bodied member arrows should end the previous line

Expression-bodied members should keep the `=>` operator at the end of the member declaration line instead of placing it at the beginning of the next line.

```csharp
// discouraged
public int GetValue()
    => 1;

// preferred
public int GetValue() =>
    1;
```

This keeps the expression-body marker attached to the member declaration and follows the Faithlife line-break style for operators.
