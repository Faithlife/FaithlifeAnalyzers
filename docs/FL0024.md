# FL0024: Expression-bodied method arrows should end the previous line

Expression-bodied methods should keep the `=>` operator at the end of the method declaration line instead of placing it at the beginning of the next line.

```csharp
// discouraged
public int GetValue()
    => 1;

// preferred
public int GetValue() =>
    1;
```

This keeps the expression-body marker attached to the method declaration and follows the Faithlife line-break style for operators.
